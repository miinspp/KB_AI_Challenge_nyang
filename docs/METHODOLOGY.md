# 산출식 (상위 % 진단) — 근거와 방법론

> 데이터 출처·기간·수집 방식은 [`DATA.md`](DATA.md), 이 문서는 **그 데이터로 점수를 어떻게 만드는지**를 다룬다.
> 구현: 백엔드 `com.nyang.rank.application.RankService`. 모든 축의 근거·한계는 `/api/rank` 응답 `notes[]`에도 실린다.

## 한눈에

```
종합점수 = (1 − 보정가중) × [ 매출 0.5 + 순수익 0.3 + 비용효율 0.2 ]      ← 기본 3축
         + 비용구조 0.15 + 매출안정성 0.15                              ← 선택 보정축(입력 있을 때만)
상위 % = 100 − 종합점수
```

- 보정축은 **입력이 있을 때만** 활성화되고, 활성화된 만큼만 기본 3축 가중치를 비례 축소한다.
- 보정 입력이 하나도 없으면 v1(기본 3축)과 **수치가 완전히 동일**하다(하위호환).
- 단위가 다른 지표를 그냥 더하지 않고, 모두 **업종 내 백분위 또는 벤치마크 대비 점수(0~100)**로 변환한 뒤 가중합한다.

## 기본 3축

| 축 | 가중 | 산출 | 데이터 |
|---|---|---|---|
| **매출** | 0.5 | 업종 매출 분포(점포수 가중 백분위 격자)에서 내 월매출의 퍼센타일을 역보간 | 서울시 상권분석서비스(실측) |
| **순수익** | 0.3 | 매출 분포 × 업종 평균 영업이익률로 **유도한** 순수익 분포에서 (매출−지출)의 퍼센타일 | 서울 분포 × 실태조사 이익률 |
| **비용효율** | 0.2 | 내 영업이익률 ÷ 업종 벤치마크 이익률 (같으면 50점, 2배 100점, 0 이하 0점) | 실태조사 이익률 벤치마크 |

- 퍼센타일 역보간: 격자 `q[0..100]`에서 값 `v`의 위치를 선형 보간. 플랫 구간은 구간 중앙값.
- 상권유형(골목/발달/전통시장)을 고르면 **같은 유형 분포**로 보조 순위(`areaRank`)를 같은 방식으로 함께 낸다.
- 한계: 순수익 분포는 업종 평균 이익률로 유도한 근사라 **점포별 이익률 편차 미반영**. 매출은 카드데이터 기반 추정치.

## 보정축 (v2)

### ① 비용 구조 (0.15) — 입력: 임대료(필수), 인건비·재료비(선택)

- **벤치마크 있을 때** (`public_benchmarks.json` 로드): 임차료율·인건비율·원가율 중 입력된 항목을
  각각 **업종 실측 평균과 비교**해 평균. `costRatioScore(내비율, 벤치마크) = clamp(50 × (2 − 내비율/벤치마크))`
  → 업종 평균이면 50점, 절반이면 75점, 2배 이상이면 0점.
- **상권유형 지역 보정**: 상권유형을 고르면 임차료율 벤치마크에 서울 상가 임대료 기반 배수를 곱한다.
  `조정 임차료율 = 전국 임차료율 × 보정배수` (골목 ×1.00 · 발달 ×1.074 · 전통 ×0.945). 상세 → [`DATA.md`](DATA.md).
- **벤치마크 없을 때** (파일 미로드): 임대료 부담률 경험칙으로 폴백 — 10% 이하 100점, 25% 이상 0점 선형.
- 근거: "매출·지출만 보면 큰 가게가 무조건 상위"가 되는 편향을, 규모 무관 정규화 지표(비율)로 상쇄.
  업종마다 정상 비용구조가 다르므로(편의점은 원가율 높고 임차료율 낮음) **업종별 벤치마크**로 채점한다.

### ② 매출 안정성 (0.15) — 입력: 최근 월별 매출 3개월 이상(홈택스 연동 시 자동)

- **추세**: 월별 매출의 OLS 기울기 ÷ 평균 = 월평균 성장률. `trendScore` = 50 + 50×(성장률/5%), ±5%/월에서 포화.
- **변동성**: 변동계수 CV = 표준편차 ÷ 평균. `volatilityScore` = 5% 이하 100점 ~ 30% 이상 0점 선형.
- 안정성 점수 = 추세 50% + 변동성 50%.
- 근거: 스냅샷(한 달 매출)이 아니라 시계열의 성장·변동을 봐야 리스크가 드러난다.

## 의도적으로 뺀 것 (데이터 근거 부재)

| 지표 | 왜 뺐나 |
|---|---|
| 평당매출·직원 1인당 매출 | 업종별 면적·직원수 분포 공공데이터 없음 → 객관적 백분위 산출 불가 |
| 부채·원리금 상환부담률 | 마이데이터 영역, 통계 분포 없음 |
| 운영 연수 보정 | 업종별 업력 곡선 데이터 미확보 |
| 전국 순위 | 매출 분포가 서울 한정. "전국 상위 X%"는 근거 없어 사용 금지 |

> 원칙: **근거를 댈 수 있는 축만 점수화한다.** 비교할 공공 분포가 없는 값은 받지 않거나 참고 표시로만 둔다.

## 하위호환·검증

- `RankService.rank`는 3단 오버로드: `(ind, req)` → `(ind, req, benchmark)` → `(ind, req, benchmark, areaMultiplier)`.
  앞 시그니처는 뒤에 null을 넘기는 위임이라, 보정 입력이 없으면 기존 결과와 동일하다.
- 단위 테스트(`RankServiceTest`)가 각 함수(퍼센타일·마진·costRatio·추세·변동성·지역보정)와 종합 합성을 개별 검증한다.
